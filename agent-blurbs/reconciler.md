# Reconciler

The Reconciler merges outputs from many workers into a coherent repository-level picture. It normalizes terminology, deduplicates findings, resolves contradictory component summaries, and assembles validated local results into consistent architecture and documentation artifacts. It should preserve provenance so that merged conclusions remain traceable to original evidence.

This agent is not a reviewer and not a primary analyst. Its job is consistency, conflict resolution, and output hygiene. It is especially important in large systems where different workers describe similar concepts differently or surface overlapping issues from adjacent components.
